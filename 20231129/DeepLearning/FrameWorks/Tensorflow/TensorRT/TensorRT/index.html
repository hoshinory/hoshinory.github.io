TensorRT版本对照表, 已验证可行

2.1 错误

2.1.1 找不到动态链接库: libnvinfer.so.8

  1. 环境
  • python3.8
  • TensorRT-8.6.1.6
  • Tensorflow2.12.0
  • cuda-12.2.2
  • cudnn-8.8.1
  1. 问题: 安装完Tensorrt以后, import tensorrt找不到 libnvinfer.so.8文件
  1. 解决办法: 在环境变量(~/.bashrc)中导入libnvinfer.so.8的地址, 关键是怎么拿到这个文件的地址, 有两种方式

python3 -c "import tensorrt; print(tensorrt.__path__)" >['/home/hoshinory/conda/envs/onnx/lib/python3.8/site-packages/tensorrt']

  1. 一些坑
  • 直接使用pip install或者conda install tensorrt可能找不到某些版本, 去TensorRT官网下载对应版本的tar包, 解压以后, 进入解压后的~/TensorRT-8.6.1.6/python 文件夹内, 找到对应版本的.whl文件, 通过pip install tensorrt-8.6.1-cp38-none-linux_x86_64.whl即可(需要对应个人的python版本)
  • cuda12.2.2可以通过Docker拉取Nvidia的官方镜像
  • 在Tensorflow官网, 目前tensorflow2.14.0推荐的cuda版本是cuda-11.6 + cunn8.6, 实际上cuda-12.2.2 + cudnn-8.8.1也能正常支持
