1.3 构建镜像(Dockerfile)

需要明白的事情:

  • docker的架构是 模式, docker并不是在本地实现build, 而是在docker的server或者engine实现build, 由此引出一个概念Context(上下文)

  • docker build -t nvidia/cuda:hello .命令中的点是指Context路径, 一般默认是使用当前路径

  • docker的build经历了: 拷贝上下文路径中的文件至server 基于server里的文件build

  • Dockerfile 不一定要放在当前目录下, 也可以不一定叫Dockerfile(什么名称都行), docker build -f FakeDockerfile .指定具体文件位置即可

需要注意的事情:

  1. 最好是新建一个空文件夹, 将build所需文件及Dockerfile放至其中
  2. 上下文路径最好不要包含多余的目录或者文件, 否则可能会上传很大的文件并且 build失败
  3. 可以使用.dockerignore 文件来排除不需要的文件(防止其上传至server)

1.3 启动镜像(容器化)

  1. -itd 表示使其挂在后台成为守护线程
  2. -p 表示将container端口与外部ip + port绑定, 就像是一个软链
    1. 个人觉得, 用外部物理机ip + port 绑定了容器暴露的端口这样比较合适
    2. 外部ip + port 容器ip + port, 外部在前, 容器在后
    3. -P 是随机分配一个 外部端口 容器
    4. -p 如果不指定ip 或者指定0.0.0.0表示将所有ip绑定
