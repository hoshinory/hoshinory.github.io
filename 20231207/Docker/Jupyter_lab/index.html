Linux下terminal环境变量在Jupyter-lab不生效问题

1. 问题还原

(1)环境

  • GPU: A100*4
  • python3.10
  • cuda-12.2.2
  • cudnn-8.8.1
  • Tensorflow2.15
  • TensorRT-8.6.1.6

The main problem is enviromental variables not worked in the notebook. It prevent me to develop codes with GPU.

There is a Infinitie loops BUG in prophet, detail shows as following:

  • All programms are running on a Linux image built by docker.

  • You must keep a main progress alive when you running a Linux image. Main progress can not be changed once you runing image. And image will exit if the main progress died.

  • Notebook is running in a Linux image as a main progress in platform of prophet. I donot kown which config file(maybe no config file, assert there is a config file) was loaded when Notebook running. What's worse, the config file loaded does not include enviromental variables we needed.

  • I search the answer in Google, and something interesting happend. >(1) I must source ~/.bashrc first and then restart the progress of notebook

    1. If I restart the progress of notebook, I must kill the progress of notebook at beginning.
    2. If I kill the progress of notebook at beginning(I remind that it is a main progress), the image will exit and restart. When Notebook auto restart, it loads config file that nobody knows for sure.
    3. If I changed main progress(someone else expcept Notebook), image cannot be running.
